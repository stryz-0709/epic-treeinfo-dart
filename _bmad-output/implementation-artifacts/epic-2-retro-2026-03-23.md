## Defects Prevented (Evidence-Based)

| Prevented defect class | Prevention mechanism | Evidence source |
|---|---|---|
| Duplicate check-ins from retries/concurrency | Unique per-day semantics + lock-protected ingest update flow | Story `2-2` completion notes/tests |
| Unauthorized check-in writes by non-ranger roles | Explicit ranger-only route dependency + role checks | Story `2-2` tasks and tests |
| Cross-ranger data leakage in summaries | Server-side role-scoped filtering + regression tests for leader/ranger paths | Story `2-1` tests and completion notes |
| Stale API response overwriting newer calendar state | Stale async request guard in provider | Story `2-4` completion notes/tests |
| Session rollover leaking previous filter/scope state | Session-scoped filter reset logic | Story `2-4` completion notes/tests |
| Runtime instability from malformed session/summary rows | Defensive claim validation + malformed-row filtering | Story `2-1` completion notes |
| Spurious check-in calls from whitespace auth tokens | Token trimming/no-op gating + regression coverage | Story `2-3` completion notes/tests |
| Incorrect pagination behavior for empty results | Empty-result pagination invariant hardening | Story `2-1` completion notes/tests |

## Process Review

### What worked well

- **Adversarial review loops were operationalized** and run to zero actionable findings before closure.
- **Targeted + full regression sequencing** was used repeatedly, reducing hidden cross-surface regressions.
- **Story artifacts captured actionable engineering evidence** (debug logs, completion notes, change logs), which improved traceability.
- **No-blame quality posture** encouraged systematic fixes over superficial patches.

### Friction observed

- Clock-mocking in check-in tests initially invalidated bearer tokens, exposing a shared test-fixture coupling risk.
- Early mobile validation attempts showed environment inconsistency around Flutter tool availability.
- Defect-prevention insights were present, but not yet normalized into one consistent artifact section template.

## Process Improvement Actions (Carry Forward)

| Action | Owner | Success criteria |
|---|---|---|
| Introduce auth-safe time-mocking test helper for check-in scenarios | Backend Dev + QA | Time-dependent tests no longer create token-validity side effects. |
| Standardize adversarial triage capture in every story artifact | Scrum Master + Dev Lead | Each story records high/medium/low findings and explicit dispositions before `done`. |
| Create shared role-scope contract test baseline for all mobile endpoints | Backend QA | New endpoint changes include self-scope, team-scope, and unauthorized-path assertions. |
| Reuse stale-request and session-reset patterns across provider-driven screens | Mobile Dev | Async state race conditions and session crossover issues are explicitly covered by provider tests. |
| Add mandatory “Defect Prevention Ledger” section in story closure template | Scrum Master | Each closed story includes at least one prevented-defect statement backed by test or code evidence. |
